System Overview
What we're building and how the pieces connect

EasyHeals Consult is a two-part product: a Conversational AI Health Chat that lives at easyheals.com/consult, and a CRM Dashboard for internal health advisors to manage leads, follow-ups, appointments and WhatsApp outreach.

ℹ️
Zero impact on existing site. The entire system lives as a new subdomain consult.easyheals.com or a path easyheals.com/consult. No existing code is touched. The CRM is a separate internal app at crm.easyheals.com.

Data Flow — Lead Capture to Conversion

1
User opens Chat
Conversational AI greets user. No login required. Language auto-detected from browser, or user picks manually.
2
Symptoms collected via chat
Free text + structured chip selections. Context stored in frontend state. No DB write yet.
3
Contact details captured in-chat
Bot asks Name → Phone → City → Preferred time. Each answer is validated inline before proceeding. On completion, POST /api/leads fires immediately.
4
Prescription upload (gated)
Upload zone unlocks ONLY after lead is saved. File goes to S3. Claude API reads it and returns structured JSON. Summary shown in chat.
5
Lead appears in CRM instantly
Health advisor sees new lead with status New, full symptom context, preferred callback time, and prescription summary if uploaded.
6
Automated WhatsApp welcome sent
Job queue fires welcome message via Meta WhatsApp Business API within 30 seconds of lead creation.
7
Advisor calls, updates status
CRM tracks call outcome, books appointment, logs notes. Follow-up reminders auto-scheduled via job queue.

Backend Services
Core service layer implementation patterns

Lead Service

jspackages/api/src/services/leadService.js
// Generate human-readable ref ID: EH-XXXXXX
async function generateRefId() {
  const last = await db.query('SELECT ref_id FROM leads ORDER BY created_at DESC LIMIT 1')
  const num = last.rows[0] ? parseInt(last.rows[0].ref_id.split('-')[1]) + 1 : 100001
  return `EH-${num}`
}

export async function createLead(data) {
  const refId = await generateRefId()
  const lead = await db.insert(leads).values({
    ...data, ref_id: refId, status: 'new'
  }).returning()

  // Fire-and-forget: WA welcome + activity log
  setImmediate(async () => {
    await logActivity(lead.id, 'lead_created')
    await waQueue.add('send-welcome', { lead_id: lead.id }, { delay: 5000 })
  })
  return lead
}

Follow-up Worker (BullMQ)

jspackages/api/src/workers/followUpWorker.js
import { Worker, Queue } from 'bullmq'

export const followUpQueue = new Queue('follow-ups', { connection: redis })

// Schedule follow-up when lead is created
export async function scheduleFollowUp(leadId, delayMs = 7200000) { // 2 hours
  await followUpQueue.add('check-lead', { leadId }, {
    delay: delayMs,
    attempts: 3,
    backoff: { type: 'exponential', delay: 300000 }
  })
}

new Worker('follow-ups', async (job) => {
  const lead = await getLead(job.data.leadId)
  if (lead.status === 'new') {
    // Still not contacted — send WA nudge + notify advisor
    await sendWhatsApp(lead, 'follow_up_check_in')
    await notifyAdvisor(lead.assigned_to, `Overdue: ${lead.name} hasn't been contacted`)
  }
}, { connection: redis })

Chat Module
State machine & conversation flow
💡
Key Design: The chat uses a simple state machine. flowStep is a number 0–N that determines which bot message to show next. Contact capture (steps 3–6) uses inline inputs directly in the chat. Prescription upload is gated — rxZoneVisible only becomes true after leadSaved === true.

Zustand Store (chatStore.js)

jsapps/chat/src/store/chatStore.js
import { create } from 'zustand'

export const useChatStore = create((set, get) => ({
  // State
  messages: [],
  chips: [],
  flowStep: 0,
  lang: 'en',
  leadData: { name: '', phone: '', city: '', time: '' },
  leadSaved: false,
  leadId: null,
  rxZoneVisible: false,
  rxAnalysis: null,
  waitingFor: null, // 'name'|'phone'|'city'|null

  // Actions
  addBotMessage: (text, chips = []) => set(s => ({
    messages: [...s.messages, { role: 'bot', text, id: Date.now() }],
    chips
  })),
  addUserMessage: (text) => set(s => ({
    messages: [...s.messages, { role: 'user', text, id: Date.now() }],
    chips: []
  })),
  setLeadSaved: (id) => set({ leadSaved: true, leadId: id }),
  setRxAnalysis: (data) => set({ rxAnalysis: data, rxZoneVisible: false }),
}))

Phone Validation (inline input)

jsapps/chat/src/components/InlineInput.jsx
export function InlineInput({ type, placeholder, validate, onSubmit }) {
  const [val, setVal] = useState('')
  const [error, setError] = useState(null)

  const handleSubmit = () => {
    const err = validate?.(val)
    if (err) { setError(err); return }
    onSubmit(val)
  }
  return (
    <div className="inline-input-wrap">
      <input type={type} value={val} onChange={e => setVal(e.target.value)}
        placeholder={placeholder} className={error ? 'error' : ''}
        onKeyDown={e => e.key === 'Enter' && handleSubmit()} />
      {error && <span className="error-hint">{error}</span>}
      <button onClick={handleSubmit}>➤</button>
    </div>
  )
}

AI & Prescription Reading
Claude API integration for prescription analysis
⚠️
Medical Disclaimer is mandatory. All AI outputs must display: "This is an AI-assisted summary for informational purposes only. It does not constitute medical advice." This must appear in the UI, be logged in the DB, and be included in any WA/email communication.

Prescription Analysis Service

jspackages/api/src/services/prescriptionAI.js
import Anthropic from '@anthropic-ai/sdk'
import { getSignedUrl } from './storageService.js'

const client = new Anthropic()

export async function analysePrescription(fileBuffer, mimeType) {
  const base64 = fileBuffer.toString('base64')

  const response = await client.messages.create({
    model: 'claude-opus-4-6',
    max_tokens: 1024,
    messages: [{
      role: 'user',
      content: [
        { type: 'image',
          source: { type: 'base64', media_type: mimeType, data: base64 }},
        { type: 'text', text: PRESCRIPTION_PROMPT }
      ]
    }]
  })

  const text = response.content[0].text
  // Claude returns JSON — strip any markdown fences
  const json = text.replace(/```json|```/g, '').trim()
  return JSON.parse(json)
}

const PRESCRIPTION_PROMPT = `
You are a medical document analyser for EasyHeals, an Indian healthcare platform.
Analyse this prescription image and return ONLY a JSON object (no markdown) with:
{
  "medications": [
    { "name": string, "dosage": string, "frequency": string, "duration": string }
  ],
  "likely_condition": string,
  "doctor_name": string | null,
  "date": string | null,
  "warnings": [string],
  "recommended_specialist": string,
  "language": "English" | "Hindi" | "Mixed",
  "confidence": "high" | "medium" | "low",
  "disclaimer": "AI-assisted summary only. Not medical advice."
}
If the image is not a prescription, return { "error": "not_a_prescription" }.
`

WA Service

jspackages/api/src/services/whatsappService.js
export async function sendTemplate(phone, templateName, variables) {
  const res = await fetch(
    `https://graph.facebook.com/v18.0/${process.env.WA_PHONE_NUMBER_ID}/messages`,
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.WA_ACCESS_TOKEN}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        messaging_product: 'whatsapp',
        to: `91${phone}`, // India prefix
        type: 'template',
        template: {
          name: templateName,
          language: { code: 'en_IN' },
          components: [{
            type: 'body',
            parameters: Object.entries(variables).map(([_, v]) => ({
              type: 'text', text: v
            }))
          }]
        }
      })
    }
  )
  if (!res.ok) throw new Error(await res.text())
  return await res.json()
}
